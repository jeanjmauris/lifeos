# operating-manual.md — How I work

The practical manual to you: strengths to lean on, blind spots to guard, and how the
assistant should work with you. Distinct from `profile.md` (identity/values, the *why*)
— this is *behaviour* (the *how*), the stuff the assistant uses mid-protocol.

**Rules for this file:**
- **Evidence-earned.** A trait belongs here once `reflections.md` backs it up — not
  because it sounded right once. Mark unconfirmed entries `[DRAFT — confirm]`.
- **Trait, not state.** A hard season, illness, or grief are *states* (they live in
  Current Season / Sensitive Context). This file is for stable patterns. Don't let a
  season get written down as a permanent trait.
- **Every blind spot gets a guardrail.** A weakness with no countermeasure is just
  self-criticism. Keep it lean and useful, never a verdict on character.

> You don't have to fill this in at setup. It earns itself over time — the `reflect`
> and `update system` protocols will propose entries as patterns emerge. Below is an
> example to show the shape; replace or delete it.
